你知道JSON.stringify有三个参数吗

发表于 2020-10-14

JSON表示一种数据格式，可以用来表示普通类型（数值字/符串/布尔值）、对象、数组、null，与XML相比更简洁便利。

我们通常使用JSON.stringify将对象序列化为json字符串，用JSON.parse将json数据解析为js对象，如下是一个简单的demo。

JSON.stringify(val, ?replacer, ?space)

实际上，JSON.stringify可以接收三个参数，分别是value(js对象), ?replacer(过滤器), ?space(缩进表示)。（?表示可选）

replacer**过滤器可以是数组或者函数形式。**

  • 数组项表示要返回的key值
  • 函数接收两个参数，key value，返回值就是相应key值，默认返回value

space缩进**可以是数字或字符**

  • 数字 表示缩进多少个空格
  • 字符 表示用此字符表示缩进

toJSON

toJSON方法允许开发者自定义序列化的行为。可以作为过滤器的补充。

序列化对象的顺序如下^[1]^

  1. 若存在toJSON方法，则调用并返回该方法的返回值，否则返回对象本身
  2. 如果存在过滤器，则调用，传入值为第一步的返回值
  3. 对第二步每个值进行序列化
  4. 如果存在相应对象，执行相应的格式化

JSON.parse(value, ?reviver)

同样，JSON.parse也接收两个参数，value(json字符串), reviver(还原函数)。这个还原函数和上面的过滤器一样，也接受两个参数key/value，只是在不同的环境下，所以换了个叫法。

不会被序列化的值

并不是所有的值都会被序列化，以下情况就会忽略掉键值对

  • value值为undefined
  • value值不可枚举
